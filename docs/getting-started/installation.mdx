## Prerequisites

## Quick Installation

Choose your installation method based on your project needs:

<Tabs>
  <Tab title="New Project">
    Start a new project with TUI-Kit-AI:
    
    <CodeGroup>
    
    ```bash npm
    mkdir my-tui-app && cd my-tui-app
    npm init -y
    npm install @tui-kit-ai/core @tui-kit-ai/ai @tui-kit-ai/agents
    npm install --save-dev typescript @types/node tsx
    ```
    
    ```bash yarn
    mkdir my-tui-app && cd my-tui-app
    yarn init -y
    yarn add @tui-kit-ai/core @tui-kit-ai/ai @tui-kit-ai/agents
    ```

    ```bash pnpm

    pnpm init
    pnpm add @tui-kit-ai/core @tui-kit-ai/ai @tui-kit-ai/agents
    pnpm add -D typescript @types/node tsx
    ```
    
    </CodeGroup>
  </Tab>
  
  <Tab title="Existing Project">

## Project Setup

### 1. TypeScript Configuration

Create a `tsconfig.json` file optimized for TUI-Kit-AI:

```json tsconfig.json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "commonjs",
    "lib": ["ES2022"],
    "declaration": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "moduleResolution": "node",
    "allowSyntheticDefaultImports": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "types": ["node", "blessed"]
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist", "**/*.test.ts"]
}
```


## Development Environment

### Terminal Configuration

For the best TUI-Kit-AI experience, configure your terminal:

<Tabs>
  <Tab title="VS Code Integrated Terminal">
    Add to VS Code settings.json:
    
    ```json
    {
      "terminal.integrated.env.osx": {
        "TERM": "xterm-256color",
        "COLORTERM": "truecolor"
      },
      "terminal.integrated.env.linux": {
        "TERM": "xterm-256color",
        "COLORTERM": "truecolor"
      }
    }
    ```
  </Tab>
  
  <Tab title="iTerm2 (macOS)">
    1. Enable true color: `Preferences > Profiles > Terminal > Report Terminal Type: xterm-256color`
    2. Enable mouse reporting: `Preferences > Profiles > Terminal > Mouse Reporting: Yes`
    3. Unicode support: `Preferences > Profiles > Text > Unicode normalization: NFC`
  </Tab>
  
  <Tab title="Windows Terminal">
    Add to settings.json:
    
    ```json
    {
      "profiles": {
        "defaults": {
          "colorScheme": "Campbell Powershell",
          "font": {
            "face": "Cascadia Code",
            "size": 12
          }
        }
      }
    }
    ```
  </Tab>
</Tabs>

### AI Provider Setup

Configure your preferred AI providers:

<AccordionGroup>
  <Accordion title="OpenAI Setup" icon="openai">
    1. Create an account at [OpenAI](https://platform.openai.com)
    2. Generate an API key in your account settings
    3. Add to your `.env` file:
    
    ```bash
    OPENAI_API_KEY=sk-your-key-here
    OPENAI_ORG_ID=org-your-org-here  # Optional
    ```
    
    4. Test the connection:
    
    ```typescript
    import { AIService } from '@tui-kit-ai/ai';
    
    const ai = new AIService({
      provider: 'openai',
      apiKey: process.env.OPENAI_API_KEY
    });
    
    const response = await ai.complete('Hello, world!');
    console.log(response);
    ```
  </Accordion>
  
  <Accordion title="Anthropic Setup" icon="anthropic">
    1. Sign up at [Anthropic](https://console.anthropic.com)
    2. Create an API key
    3. Add to your `.env` file:
    
    ```bash
    ANTHROPIC_API_KEY=sk-ant-your-key-here
    ```
    
    4. Test the connection:
    
    ```typescript
    const ai = new AIService({
      provider: 'anthropic',
      apiKey: process.env.ANTHROPIC_API_KEY
    });
    ```
  </Accordion>
  
  <Accordion title="Ollama Setup (Local)" icon="server">
    1. Install Ollama from [ollama.com](https://ollama.com)
    2. Pull a model:
    
    ```bash
    ollama pull llama2
    ```
    
    3. Start the service:
    
    ```bash
    ollama serve
    ```
    
    4. Configure in your application:
    
    ```typescript
    const ai = new AIService({
      provider: 'ollama',
      baseURL: 'http://localhost:11434',
      model: 'llama2'
    });
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Module Not Found Errors">
    **Problem**: `Cannot find module '@tui-kit-ai/core'`
    
    **Solutions**:
    1. Verify installation: `npm list @tui-kit-ai/core`
    2. Clear cache: `npm cache clean --force`
    3. Reinstall: `rm -rf node_modules package-lock.json && npm install`
    4. Check Node.js version: `node --version` (should be 18+)
  </Accordion>
  
  <Accordion title="Terminal Rendering Issues">
    **Problem**: Strange characters or broken layouts
    
    **Solutions**:
    1. Check terminal capabilities: `echo $TERM`
    2. Force color mode: `export FORCE_COLOR=3`
    3. Use compatible terminal (VS Code, iTerm2, Windows Terminal)
    4. Update terminal settings for Unicode support
  </Accordion>
  
  <Accordion title="TypeScript Compilation Errors">
    **Problem**: Type errors or compilation failures
    
    **Solutions**:
    1. Install type definitions: `npm install --save-dev @types/blessed`
    2. Update TypeScript: `npm install --save-dev typescript@latest`
    3. Check tsconfig.json configuration
    4. Restart TypeScript server in your IDE
  </Accordion>
  
  <Accordion title="AI Provider Connection Issues">
    **Problem**: API connection failures
    
    **Solutions**:
    1. Verify API keys are correct
    2. Check network connectivity
    3. Test with curl:
    ```bash
    curl -H "Authorization: Bearer $OPENAI_API_KEY" \
         https://api.openai.com/v1/models
    ```
    4. Check rate limits and billing status
  </Accordion>
</AccordionGroup>
